小蚂蚁后台数据文档

任务 Entity

字段 类型 是否必需 说明
taskInd int 否 任务id
userId int 否 任务创建者id
taskType int 是 任务类型
taskTitle string 是 任务标题/任务名称
taskDesc string 是 任务描述
taskCount int 是 任务总数
taskFee double 是 任务价格
taskTemplate json 是 任务具体内容
deadLine date 是 截止日期
availableTime date 是 任务上线时间
createTime date 否 创建时间由后台生产，前端不用管
taskRes string 否 任务来源
priceType int 是 价格类型。{1：人民币,2:金币}
limitCity string 是 限制城市
availableTime date 是 上线时间

日期格式为：「yyyy-MM-dd HH:mm:ss」2015-09-11 21:08:20

个人任务 Entity

字段 类型 是否必需 说明
pInd int 否 个人任务id
taskInd int 是 任务id
userId int 是 个人任务创建者id
checkUserId int 是 个人任务审核者
addInd int 是 子任务列表
taskContent json 是 个人任务内容
submitTime date 否 提交时间
checkDesc string 是 审核描述
checkTime date 是 审核时间
checkStatus int 是 任务审核状态 {0：待完成，1：待审核，2：审核通过，3：未通过}
makeMoney double 是 个人任务价格
priceType int 是 价格类型。{1：人民币，2：金币}
city string 是 提交任务城市

以下查看列表接口 默认都有参数 pageNo=1，和pageSize=20

任务结构体里的几个元数据实体，伪代码 图片：

private String[] value={};

private String type="img";
private int num=1; //照片数量

单选：

private String type="radio";
private String[] value={"刘德华","斯嘉丽约翰逊"};

多选：

private String type="checkbox";
private String[] value={"刘德华","成龙","威尔史密斯"};

文本：

private String value="";
private String type="text";
private boolean required=true; //是否必填
private int length=15; //限制长度

空间：

private String type="geometry";
private String value="";  
   private String location =Location.GPS.value; //  gps,mixed,disable

他们的上一级是 questionEntity

 private String name=""; //即问题。eg：你的性别
private JSONArray content =new JSONArray(); //元数据数组。如 { "content":[{"type":"geometry","value": ""}],"filed": "location","name": "获取GPS定位信息"}

    private String filed;

任务的结构体为 questionList：[{questionEntity}]

任务编辑

POST http://back.antzb.com/web/tasktemplate/edit.do
表单提交的参数为 ：
taskTemplateId =12；
taskTitle=任务标题;
taskDesc=任务描述;
taskCount=10 //任务总数；
price=1 //价格；
priceType=1 //价格类型 {1，2}；
processLimit=1 //执行类型，多个值为20;avaliableTime=xxx  //上线日期；
deadLine=xxx //截至日期

可能的返回结果

    {"code":200,"message":success}

审核任务

POST : back.antzb.com/web/personaltask/audit.do

提交的参数：
eg 审核893 为不通过

pind=893    
checkStatus=3
checkdesc=审核不通过。    

审核894 为通过

pind=894
checkStatus=2

服务器返回

{"code":200,"message":"success"}

任务审核列表

该接口是只要在个人任务列表接口 多加一个参数就好；

eg : back.antzb.com/web/personaltask/items.do?taskInd=20
